# 创建订单页颜色尺码联动设计文档

## 背景

当前创建订单页面的订单明细仍使用“规格”自由输入字段，无法和成品库存中的颜色、尺码、成本信息建立明确对应关系。

这带来几个问题：

- 订单录入时无法按库存中的真实颜色和尺码组合选择商品
- 同一产品在成品库存中存在多个颜色、尺码时，当前页面缺少联动筛选能力
- 订单详情页和销售归档页只能展示模糊的“规格”文本，不能准确回显颜色和尺码
- 订单创建时无法把当前选中商品对应的单件成本同步展示给用户

## 目标

- 将创建订单页中的“规格”字段替换为“颜色”和“尺码”两个下拉框
- 产品、颜色、尺码三者基于成品库存数据实现双向联动筛选
- 当产品、颜色、尺码唯一命中某条成品库存记录时，自动展示该记录的单件成本
- 颜色和尺码正式保存到订单数据中
- 订单详情页和销售归档页将“规格”展示替换为“颜色 + 尺码”

## 非目标

- 单件成本不参与订单金额计算，不写入订单数据
- 不新增新的后端联动查询接口，优先复用现有成品库存列表数据
- 不调整订单审批、发货、完成等既有业务流程

## 现状分析

### 当前创建订单页

文件：`frontend/src/views/order/Create.vue`

当前页面特点：

- 产品下拉直接使用 `/inventory/finished-products` 列表数据
- 订单项结构为 `productId + productName + specification + quantity + unitPrice + amount`
- `productId` 当前直接取成品库存记录 ID
- 变更产品时只会同步 `productName` 和 `unitPrice`

### 当前后端订单结构

文件：

- `src/main/java/com/garment/dto/OrderItemDTO.java`
- `src/main/java/com/garment/model/OrderItem.java`
- `src/main/java/com/garment/service/impl/OrderServiceImpl.java`

当前问题：

- 订单项正式存的是 `specification`
- 后端创建订单时用 `productId` 去 `ProductDefinitionRepository` 查产品编号
- 但前端传入的 `productId` 实际上是成品库存记录 ID，不是产品定义 ID

### 当前详情与销售归档展示

文件：

- `frontend/src/views/order/Detail.vue`
- `src/main/java/com/garment/model/SalesRecord.java`
- `src/main/java/com/garment/dto/SalesRecordVO.java`
- `frontend/src/views/sales/Record.vue`

当前详情页和销售归档页都依赖 `specification` 字段展示规格信息。

## 方案选择

### 方案一：前端使用成品库存列表作为唯一联动数据源

做法：

- 创建订单页一次性拉取成品库存列表
- 前端本地计算“产品名-产品编号”的唯一产品选项
- 根据已选产品、颜色、尺码在前端计算可选项与唯一命中记录

优点：

- 与当前页面数据来源一致
- 不需要新增后端联动接口
- 交互实时，页面响应更直接
- 改动面可控

缺点：

- 前端联动逻辑会变复杂一些

### 方案二：新增后端订单选品联动接口

做法：

- 后端提供产品、颜色、尺码联动专用接口
- 前端按用户选择动态请求可选值

优点：

- 筛选规则集中在后端

缺点：

- 这次需求范围会明显扩大
- 当前页面已经拉取了成品库存列表，新增接口收益不高

### 方案三：继续以成品库存单条记录直接作为产品选项

做法：

- 产品下拉继续展示所有成品库存记录
- 颜色、尺码在选中后再做补充

优点：

- 改造最少

缺点：

- 同一产品会出现大量重复项
- 与“先选任意一个，再筛另两个”的目标不匹配
- 用户体验较差

### 推荐方案

采用方案一。

理由：

- 最符合当前页面和库存模块的既有结构
- 能实现你要求的双向联动
- 不需要为了本次功能额外扩张后端接口

## 详细设计

### 一、创建订单页交互改造

文件：`frontend/src/views/order/Create.vue`

#### 1. 订单明细列调整

原列：

- 产品
- 规格
- 数量
- 单价
- 金额

调整后：

- 产品
- 颜色
- 尺码
- 数量
- 单价
- 单件成本
- 金额

其中：

- “颜色”与“尺码”均为下拉框
- “单件成本”为只读展示字段
- “规格”列删除

#### 2. 产品下拉展示规则

产品下拉不再直接展示全部成品库存记录，而是按以下维度去重：

- `productName`
- `productCode`

展示文案格式保持为：

- `产品名-产品编号`

例如：

- `T恤-n1`

#### 3. 联动筛选规则

联动数据来源为当前已加载的成品库存列表。

每一行订单项维护以下筛选条件：

- 产品名/产品编号组合
- 颜色
- 尺码

联动原则：

- 先选产品后，颜色和尺码只显示该产品在成品库存中存在的可选值
- 先选颜色后，产品和尺码只显示仍有库存记录可匹配的可选值
- 先选尺码后，产品和颜色只显示仍有库存记录可匹配的可选值
- 当选定产品 + 颜色后，尺码继续缩小到对应组合
- 当选定产品 + 尺码后，颜色继续缩小到对应组合
- 当选定颜色 + 尺码后，产品也只显示对应组合下存在的产品

#### 4. 唯一命中规则

当一行订单项的已选条件最终唯一命中某一条成品库存记录时：

- `productId` 绑定为该成品库存记录 ID
- `productName` 同步为该记录的产品名
- `productCode` 同步为该记录的产品编号
- `color` 同步为该记录颜色
- `size` 同步为该记录尺码
- `costPrice` 在页面上展示为该记录的单件成本

#### 5. 未命中或不唯一命中的处理

当某一行尚未唯一命中成品库存记录时：

- 不展示有效的单件成本
- 不允许该行作为有效订单项提交
- 若某个已选值因联动变化而失效，应自动清空
- 清空失效字段后重新计算剩余可选项

#### 6. 金额计算

订单金额规则保持不变：

- `amount = unitPrice * quantity`

单件成本只做展示，不参与订单金额、总金额计算。

### 二、前端订单项数据结构调整

创建订单页中的单行订单项从：

```js
{
  productId: '',
  productName: '',
  specification: '',
  quantity: 1,
  unitPrice: 0,
  amount: 0
}
```

调整为：

```js
{
  productId: '',
  productCode: '',
  productName: '',
  selectedProductKey: '',
  color: '',
  size: '',
  quantity: 1,
  unitPrice: 0,
  costPrice: null,
  amount: 0
}
```

说明：

- `selectedProductKey` 用于前端联动时标识“产品名-产品编号”组合
- `costPrice` 仅前端展示使用，不提交后端

## 校验规则

### 创建订单页校验

每一行订单项必须满足：

- 选择了产品
- 选择了颜色
- 选择了尺码
- 能唯一命中成品库存中的某条记录
- 数量大于等于 1

若未满足：

- 阻止提交
- 提示用户补全颜色和尺码选择

## 边界处理

### 1. 颜色或尺码为空的历史库存数据

如果某条成品库存记录缺少颜色或尺码：

- 在联动中将空值视为不可选项
- 创建订单页不允许使用这类记录完成本次联动下单

原因：

- 这次需求明确要求订单项正式保存颜色和尺码
- 不完整库存记录不应再通过旧“规格”路径绕过选择

### 2. 联动后值失效

例如：

- 先选产品 A
- 再选颜色 红色
- 再改产品为 B

如果“红色”对产品 B 不再合法：

- 自动清空颜色
- 自动清空尺码
- 自动清空单件成本
- 重新计算可选项

### 3. 单件成本为空

若唯一命中的成品库存记录 `costPrice` 为空：

- 页面单件成本显示为空或 `-`
- 不阻止下单

原因：

- 单件成本只是辅助展示，不是订单创建必填项

## 测试设计

### 前端联动测试

至少覆盖以下行为：

1. 先选产品，再正确过滤颜色和尺码
2. 先选颜色，再正确反向过滤产品和尺码
3. 先选尺码，再正确反向过滤产品和颜色
4. 选满产品 + 颜色 + 尺码后，能唯一命中并展示单件成本
5. 修改已选条件后，失效字段会被清空

### 后端订单测试

至少覆盖以下行为：

1. 创建订单时能保存 `productCode`、`color`、`size`
2. 查询订单详情时能返回 `color`、`size`
3. 完成订单后归档销售记录时，销售记录明细中包含 `color`、`size`

### 页面展示测试

至少覆盖以下行为：

1. 订单详情页不再显示“规格”，改为“颜色”“尺码”
2. 销售归档详情页不再显示“规格”，改为“颜色”“尺码”
3. 创建订单页中“单件成本”只展示，不参与金额计算

## 验收标准

- 创建订单页中的“规格”字段被替换为“颜色”和“尺码”下拉框
- 产品下拉显示“产品名-产品编号”的唯一组合
- 产品、颜色、尺码三者支持双向联动筛选
- 唯一命中成品库存记录后可自动展示单件成本
- 创建订单后，订单数据正式保存颜色和尺码
- 订单详情页展示颜色和尺码
- 销售归档页展示颜色和尺码
- 单件成本仅在创建订单页展示，不写入订单数据
